# Daily Workflow

The day-to-day sequence. For the form you actually fill out while doing this, see the **`research-entry`** GitHub Issue template (Issues → New Issue → Research Entry) — it mirrors these phases as fields.

## Before starting

Begin with a completely fresh research cycle — see [Research Workflow](09-research-workflow.md#fresh-research-every-time). No reused scans, drafts, or market data.

## Phase 1 — Market Scan & Story Selection

Search the entire crypto market: market structure, regulation, security, on-chain activity, institutional adoption, infrastructure, stablecoins, macro events. Rank candidates using the **six criteria in [Decision Framework](06-decision-framework.md#the-six-evaluation-criteria)** — Evidence Strength, Educational Value, Market Impact, Originality, Shelf Life, Mechanism Quality. Don't rank informally; use the actual six dimensions.

## Phase 2 — Verification

Follow [Sources & Research Priority](01-sources-and-priority.md) and classify findings with [Evidence Framework](02-evidence-framework.md). Never confuse repeated reporting with independent evidence.

## Phase 3 — Explain the Mechanism

Before why it matters, explain how it happened. See [Writing & Mechanism](03-writing-and-mechanism.md).

## Phase 4 — Build the Draft

Standalone tweet first; thread only if a genuine attempt to compress fails. See [Writing & Mechanism](03-writing-and-mechanism.md#format).

## Phase 5 — Visual Assets

Retrieve supporting visuals per the single priority list in [Visual Assets](04-visual-assets.md) — four tiers, no more, no fewer. Every visual states its decision value.

## Phase 6 — QA Gate

Run the ten questions in [QA Gate](08-qa-gate.md). Any "No" → revise or set status to WAIT.

## Final Output

Every response ends with: Publication Status, Confidence Level, Evidence Classification, and any Verification Limitations.
