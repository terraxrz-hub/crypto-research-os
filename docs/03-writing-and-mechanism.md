# Writing & Mechanism

> This page was empty in the original framework, even though "mechanism before conclusion" was cited by seven other pages. This is the canonical, written version — everything else now links here instead of restating it.

## Governing principle

**The confidence in the writing must match the confidence in the evidence.**

Everything below is a consequence of that one rule, not an independent instruction:

- **Mechanism before conclusion** — explain *how* something happened before explaining *why it matters*. A reader who understands the mechanism can evaluate the conclusion themselves; a reader who only gets the conclusion has to trust you.
- **Facts before analysis** — state what was verified before interpreting it. Don't let interpretation blur into the fact record.
- **Preserve uncertainty** — if the evidence is Tier 3–4 (see [Evidence Framework](02-evidence-framework.md)), the writing must read like Tier 3–4 evidence. Don't launder a "According to Arkham" into a flat statement of fact.
- **No hype** — confident language is only earned by confident evidence. If the evidence tier is Medium, the language is Medium.

These aren't four rules to separately satisfy — they're four symptoms of the same discipline. If you're checking whether a draft is compliant, check the governing principle first; the four bullets are diagnostics, not a checklist to pass individually.

## Format

- **Standalone tweet is the default.** Attempt to compress the mechanism and conclusion into one tweet, under 280 characters, before considering a thread.
- **Thread only after a genuine attempt to fit it in one tweet fails**, or a thread is explicitly requested.
- If a thread is used: each tweet stays under 280 characters, splits naturally, never repeats information already stated, and still preserves uncertainty.

## Checklist before drafting

- [ ] Do I know the mechanism, not just the headline?
- [ ] Does my language match my evidence tier?
- [ ] Have I separated fact from analysis?
- [ ] Is uncertainty visible, not smoothed over?
