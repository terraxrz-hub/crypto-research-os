# Visual Assets

Visuals must improve understanding or verification — never just decorate the post. Every suggested visual needs a stated decision value.

## Priority (maps directly onto the [Evidence Framework](/frameworks/FRAMEWORK-001-evidence-tier-framework.md) tiers — do not maintain a separate count)

1. **Primary** — court filings, government/regulatory documents, official announcements, blockchain explorer pages (Tier 1 evidence).
2. **Trusted secondary** — official dashboards and analytics platforms: TradingView, CoinGlass, SoSoValue, Glassnode, CryptoQuant, Dune, Token Terminal, DeFiLlama, Arkham, Nansen, Lookonchain, Kaiko, Messari (Tier 2–3 evidence).
3. **Illustrative** — a self-made diagram (timeline, flowchart, process diagram) explaining *already-verified* facts. Label clearly as illustrative, never as evidence.
4. **Decorative** — visual appeal only. Avoid unless it genuinely improves the post.

> If you're counting steps here and getting a different number than the list above, stop — this page and [Daily Workflow](/playbooks/PLAYBOOK-0002-daily-workflow.md) previously drifted apart (one had 4 tiers, the other split out a 5th). There is one list now. It lives here.

## Every visual must state

- Source
- Evidence level (Primary / Trusted Secondary / Illustrative / Decorative)
- Why it improves understanding or verification

## Rules

- **Retrieve, don't just recommend**, whenever possible. If retrieval isn't possible, say so explicitly.
- **Never fabricate** screenshots, charts, or documents. Never present AI-generated visuals as evidence.
- If no trustworthy visual exists: say "No supporting visual recommended."
