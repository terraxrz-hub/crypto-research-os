# Playbooks

**What belongs here:** ordered, step-by-step procedures — the *sequence* in which Rules and Frameworks get applied to produce a Research object. A Playbook is operational, not evaluative: it doesn't itself define what "good evidence" means (that's FRAMEWORK-001), it defines *when* to check for it.

**Why this category exists:** separating sequence from judgment means the sequence can change (e.g. adding a new phase) without touching the Frameworks it invokes, and a Framework's criteria can be refined without renumbering any Playbook's phases. This is also the layer a future AI agent executes against directly — "what do I do next" is a Playbook question; "how do I judge what I find" is a Framework question.
